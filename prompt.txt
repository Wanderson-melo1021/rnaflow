Estou desenvolvendo um projeto chamado rnaflow, uma pipeline modular de RNA-seq com foco didático e arquitetura séria.

Contexto geral do projeto:
- Quero uma pipeline alignment-based, sem pseudoalignment.
- A orquestração principal deve usar C como núcleo.
- Bash deve ser apenas launcher e scripts auxiliares.
- Perl deve ser usado para validação e preparação de metadados.
- R e Python serão usados nas etapas futuras de análise estatística e relatório.
- A pipeline começa com um samplesheet TSV contendo SRRs e metadados.
- O layout SE/PE deve ser detectado automaticamente a partir dos metadados do projeto, e não manualmente.
- O download dos metadados do BioProject deve ser feito pela própria pipeline, não por mim.
- Os FASTQ devem ser comprimidos imediatamente com pigz após o fasterq-dump, para economizar espaço.
- Os arquivos SRA só devem ser apagados depois que os FASTQ comprimidos forem validados minimamente com sucesso.
- Estou limitado por espaço em disco, então a pipeline precisa processar amostra por amostra e comprimir logo após gerar os FASTQ.

Problema atual:
Quando rodei make run, recebi este erro:
[INFO] Processing sample=GSM8014143 srr=SRR27532983 layout=unstranded
[ERROR] Unknown layout 'unstranded' for GSM8014143

Isso aconteceu porque há desalinhamento entre os schemas dos arquivos/scripts.
O script 02_data_acquisition.sh estava esperando 7 colunas:
sample condition replicate srr bioproject layout strand
mas o prepared_samplesheet.tsv gerado nessa execução tinha 6 colunas:
sample_id condition replicate srr layout strandedness

Ou seja, layout estava sendo lido na posição errada, e o valor de layout virou "unstranded".
